Acceptable Use Policy

anvil is a tool for training and experimenting with small language models. We believe in open, responsible AI development. This policy defines the boundaries of acceptable use for the software itself, bundled data, user-uploaded content, and all data that enters the system.

Universal No-Harm Stance
Every dataset, corpus, model, and training run processed through anvil must be used in a manner that does not cause harm. This is not optional. By using anvil, you accept this obligation.

Prohibited Uses

You may not use anvil, its bundled data, or any content processed through it for the following purposes:

  • Harmful or deceptive generation — creating content intended to harass, defraud, manipulate, or cause physical or psychological harm to individuals or groups.
  • Weaponisation or surveillance — using models or data for weapons development, military targeting, mass surveillance, or human-rights violations.
  • Disinformation campaigns — generating misleading content at scale intended to deceive the public, influence elections, or undermine democratic processes.
  • Child safety violations — any use involving or enabling child sexual abuse material (CSAM) or the exploitation of minors.
  • Unauthorised data collection — scraping, ingesting, or processing personal data without lawful consent or a legitimate basis under applicable privacy law.
  • Discrimination or profiling — using the system to build tools that discriminate on the basis of race, ethnicity, gender, religion, disability, sexual orientation, or age.

Data Governance Requirements

All data that enters the anvil system — whether uploaded, imported, or generated — must satisfy the following requirements:

License Declaration

Every dataset must have a declared license. If the content is your own original work, select "Own / private / proprietary content" from the license selector when uploading. For third-party content, select the applicable open-source license and provide attribution.

No-Harm Affirmation

Before any data enters the system, you must affirm that the content is licensed for training use and will not cause harm. This affirmation is recorded alongside the upload metadata.

Source Attribution

Declare the source or origin of your dataset. Whether it is a public corpus, a private collection, or generated content, a clear source declaration ensures traceability and accountability.

Bundled & Demo Data

Bundled demo datasets shipped with anvil are provided under permissive licenses (CC0, MIT) for evaluation and educational use only. You are responsible for verifying that your use of any bundled data complies with its respective license terms. Bundled data carries the same no-harm obligation as user-uploaded content.

System Usage

The anvil web interface and API are intended for local or private-network use. If you expose anvil to the internet, you are responsible for:

  • Implementing appropriate access controls and authentication.
  • Ensuring that any user of the exposed instance complies with this policy.
  • Logging and monitoring usage to detect policy violations.

Reporting Violations

If you become aware of a violation of this policy — whether through anvil's bundled data, user uploads, or system misuse — please report it via the project's issue tracker. We take all reports seriously and will investigate promptly.

Policy Scope
This policy applies to all versions and distributions of anvil, including source builds, pip-installed packages, container images, and any derivative works that incorporate anvil's code or bundled data. It may be updated from time to time; continued use after updates constitutes acceptance.
