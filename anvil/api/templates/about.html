About anvil

anvil is an interactive workbench for training and experimenting with small language models from scratch. It pairs a zero-dependency training engine — with RoPE, SwiGLU MLP, and RMSNorm, the same primitives as production Llama models — with a polished web UI, MLflow experiment tracking, and interactive learning content.

Every concept here scales up directly to real models. Configure a model, pick your data, and watch it learn character by character in real time — then dive into the interactive lessons to see what's happening under the hood. Built with 🔨 by Josh Burt.

Technology Stack

Training Engine

Pure-Python, stdlib-only engine (anvil/core/) with a custom autograd graph. Optional PyTorch backend for CUDA/MPS GPU acceleration. Zero runtime dependencies for the core.

Web Framework

FastAPI + Jinja2 templates + Server-Sent Events for live training metrics. Async SQLAlchemy with SQLite (WAL mode) for metadata persistence. MLflow for experiment tracking.

Frontend

Zero JavaScript libraries. Native EventSource, IntersectionObserver, Canvas API, and CSS custom properties with an iOS-inspired design system. Dark mode default with theme toggle.

Distribution

Pip-installable Python package (PEP 621). Docker multi-stage builds. CI via GitHub Actions with automated semantic versioning.

Architecture Overview

The codebase follows strict layered architecture: Repository → Service → God Class (AnvilWorkbench) → Routes/CLI. No shortcuts.

  • Core — Zero-dependency training engine with Llama-class architecture (RoPE, SwiGLU, RMSNorm).
  • Services — Domain-decomposed business logic: training, datasets, inference, tracking, governance, demo.
  • API — FastAPI with versioned v1 router. Jinja2 pages, SSE streaming, REST endpoints.
  • Storage — SQLite via async SQLAlchemy + LocalFileStore (S3-ready abstraction).
  • Governance — Hash-chained audit trail, acceptable-use gate, provenance tracking, license catalog.

See the project README for the full architectural breakdown.

Governance & Licensing

anvil includes a built-in governance framework for responsible data handling. Every dataset that enters the system passes through an acceptable-use gate, and all operations are recorded in a hash-chained audit trail.

Acceptable Use

Data Provenance

Every dataset and corpus in the system carries provenance metadata: source description, license identifier, attribution text, and origin (bundled or user-uploaded). Bundled demo data provenance is validated against the approved license catalog at startup.

Bundled Demo Data

Demo samples include works from Project Gutenberg (public domain), MIT-licensed name lists, and project-generated content. Each item is declared in a machine-readable provenance manifest with source, license, and attribution.
