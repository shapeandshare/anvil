Frequently Asked Questions

Everything you’ve wanted to ask about how this tiny transformer works, what it can do, and why it behaves the way it does.

Common questions about how anvil works and what it can (and cannot) do.

Does the model actually understand anything? [+]

No. The model has no understanding, consciousness, or intent. It is a statistical pattern matcher: given a sequence of characters, it predicts the next most likely character based on patterns in its training data. When it generates something that looks meaningful, it is reproducing statistical regularities, not thinking. This is sometimes called a "stochastic parrot."

Why does it hallucinate? [+]

Hallucination is just the model generating plausible-looking text that happens to be factually wrong. Since it has no understanding of truth, it simply produces whatever sequence of characters is statistically likely. If the training data contained patterns like "Einstein was born in 1879," the model may reproduce that. But it can just as easily generate "Einstein was born in 1923" if that fits a statistical pattern. Every output is just a grammar exercise, not a factual statement.

How is this related to ChatGPT? [+]

This is the same fundamental architecture: a decoder-only transformer with self-attention, trained to predict the next token. The differences are scale: ChatGPT has billions of parameters, was trained on trillions of tokens, and used thousands of GPUs for weeks. anvil has 4,192 parameters, trains on kilobytes of text in seconds, and fits on a single CPU. Every concept here scales up directly to production models — just with 10,000,000× more data and compute.

Why is it so slow? [+]

The core engine is pure Python with no GPU acceleration. Every forward pass loops through scalar operations (add, multiply, softmax) one value at a time. Production models use GPU kernels that process thousands of operations in parallel. The slowness is intentional: it lets you inspect every intermediate value and watch the computation unfold step by step — something impossible with optimised frameworks like PyTorch.

Can I make it generate better names? [+]

Better names require better data or more training. The default dataset is a small collection of medieval guild names. You can upload your own .txt dataset through the Data tab and retrain. Training for more steps, increasing n_embd, or adding n_layer also helps the model pick up longer-range patterns. But with 4,192 parameters, it will never produce genuinely creative or coherent names — it can only recombine patterns it has seen.

What if I change the dataset? [+]

The model learns whatever character-level patterns exist in the training text. Give it Shakespeare and it will generate Shakespeare-like character sequences. Give it Python code and it will generate plausible-looking code (though with only 4,192 parameters it won't compile). The vocabulary is built from the unique characters in the dataset, so a dataset with more diverse characters (uppercase, digits, punctuation) gives the model more tokens to work with but spreads its limited capacity thinner.
