Understanding Datasets & Corpora

▼
anvil provides two mechanisms for sourcing training data: datasets and corpora. Each serves a distinct purpose. Understanding when to use which is key to an efficient workflow.
What are Datasets?

Datasets are static collections of text samples where each line in a .txt file becomes one training sample. Upload plain-text files through the upload form, or create an empty dataset and add content programmatically.

Pros:

  • Inline editing — rename, update descriptions, curate content (dedup, filter, replace)
  • Simple upload workflow — upload any .txt file directly
  • Ideal for small-to-medium curated collections, fine-tuning data, or evaluation sets

Cons:

  • No automatic directory scanning — you must upload or create manually
  • Not suited for large code repository ingestion
What are Corpora?

Corpora are dynamic directory sources scanned with glob patterns and chunking strategies. Point the wizard at a directory, configure include/exclude patterns, and the system auto-detects optimal chunking parameters based on file size and type distribution.

Pros:

  • Auto-scan directories — discover files without manual upload
  • Glob pattern filtering — include/exclude by file extension or path (e.g. *.py, !tests/)
  • Chunking strategies — windowed, file, or line for different use cases
  • Gitignore-style pattern support for code repos

Cons:

  • Read-only from source — no inline sample editing
  • Requires a stable directory path
When to Use Which

Use datasets when you have curated text you want to edit, filter, or deduplicate. Fine-tuning data, evaluation benchmarks, and small-to-medium training collections belong in datasets.

Use corpora when you want to scan existing code repositories, documentation trees, or large directory structures. Corpora excel at discovering files and chunking them optimally for training.

Best Practices
  • Keep datasets small and curated — they are meant for quality-controlled training data
  • Use corpora with appropriate chunking for code repos — windowed with small overlap for source files, file for markdown docs
  • Combine corpora imports into datasets for final training — import corpus documents into a dataset to get the best of both worlds
  • Use the All Data table below to see everything at a glance and navigate quickly
