# Implementation Plan: UX Rules Integration

**Branch**: `025-ux-rules-integration` | **Date**: 2026-06-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from UX rules integration handoff (docs/usability/HANDOFF.md)

## Summary

Integrate the UX review/generation ruleset and its deterministic gate into the host repo. Place 6 existing artifacts: ruleset (`docs/ux-rules.md`), two OpenCode skills (`.opencode/skills/ux-review/SKILL.md`, `.opencode/skills/ux-generate/SKILL.md`), two Python checkers (`scripts/ci/ux_lint.py`, `scripts/ci/ux_review.py`), a Makefile include (`shared/ux.mk`), and wire Spec Kit governance via a constitution principle. All artifacts exist and are pre-tested — this feature is pure placement + wiring.

## Technical Context

**Language/Version**: Python 3.11+ (existing repo convention)  
**Primary Dependencies**: Stdlib only — `ux_lint.py` (re/ sys/ os/), `ux_review.py` (stdlib + urllib for OpenAI-compatible API calls)  
**Storage**: N/A — flat files on disk  
**Testing**: pytest (existing repo). Existing scripts carry their own inline tests (HANDOFF confirms ux_lint.py tested, ux_review.py py-compiled). New tests for the integration (Makefile targets, skill loading, constitution check) to follow.  
**Target Platform**: macOS/Linux (developer machines + CI runners)  
**Project Type**: Tooling integration (file placement + Makefile wiring)  
**Performance Goals**: Linter <2s per file (regex-based); review at model API speed  
**Constraints**: Zero pip dependencies for linter; stdlib-only for review script; all artifacts repo-internal, no network fetch  
**Scale/Scope**: Single repo, ~6 placed files, 2 Makefile targets, 1 constitution amendment
